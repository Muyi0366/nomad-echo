## 🔧 技术实现细节

### 1. 数据库设计

**insights_data.json** - 感悟库
```json
[
  {
    "text": "感悟内容",
    "emotion": "✨ 灵感",
    "timestamp": "2025-01-27 10:30:45"
  }
]
```

**chats.json** - 聊天库
```json
{
  "user_123_user_456": {
    "participants": ["user_123", "user_456"],
    "messages": [...]
  }
}
```

### 2. 海报生成流程

```python
对话消息 
  → extract_keywords()      # 关键词提取
  → generate_vibe_poster()   # 海报生成
  → 显示图片 + 配文         # 用户展示
```

### 3. 关键词提取算法

- 正则分词：`[\u4e00-\u9fa5]{2,}` 匹配 2+ 个中文字
- 停用词过滤：移除"的、是、在、了"等虚词
- 频率排序：选择出现次数最多的 3 个词

### 4. 两种海报模式

| 功能 | 本地版 | API 版 |
|------|--------|--------|
| 生成速度 | <1s | 5-30s |
| 成本 | $0 | $0-9/月 |
| 创意性 | 固定设计 | 每次不同 |
| API Key | 不需要 | 需要 |

## 🐛 已知限制和解决方案

### 限制 1：Streamlit 的刷新机制
- **问题**：Streamlit 是静态刷新，不是真正实时
- **解决**：使用 streamlit-autorefresh，每 3 秒轮询一次

### 限制 2：多用户场景
- **当前**：单浏览器实例（虚拟对话伙伴）
- **未来**：可集成 WebSocket 实现真正多人聊天

### 限制 3：API 超时
- **当前**：设置 120 秒超时
- **解决**：自动降级到本地版本
